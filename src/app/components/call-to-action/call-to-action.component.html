Call to Action

While a call to action looks like a button, it's really more of a design concept that can be applied to an anchor element or a button element, depending on its functionality. Calls to action can be any one of three styles (default, assertive, or ghost) and three sizes (default, small, or large).

Accessibility

  • Use an anchor element if the call to action navigates users to another location — keep in mind that anchor elements cannot be disabled!
  • Use a button element if the call to action performs an action on the page, such as opening a modal or submitting a form
  • Don't use the input element to create a call to action — the button element is more semantic and supports the custom focus indicator, which is required for WCAG compliance
  • Validate keyboard interactions

    • Button element keyboard interactions

Styles & Sizes

Default

Default calls to action are a secondary style. They are the default call to action style unless otherwise specified.

Assertive

Assertive calls to action can be used once per page, for the most important action. If there is insufficient contrast between the assertive call to action and the background, use a ghost call to action instead.

Ghost

Ghost calls to action should only be used on dark backgrounds that provide sufficient contrast.
